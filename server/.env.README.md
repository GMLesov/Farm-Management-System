# Farm Management System - Server Environment Variables

## Quick Setup

1. Copy this file and rename it to `.env`
2. Update the values below with your actual configuration
3. Never commit `.env` to version control

## Configuration

### MongoDB Connection
Replace `username`, `password`, and cluster address with your MongoDB Atlas credentials:
```
MONGODB_URI=mongodb+srv://your-username:your-password@your-cluster.mongodb.net/farm-management?retryWrites=true&w=majority
```

### JWT Secret
Generate a strong random string (minimum 32 characters):
```bash
# On Linux/Mac
openssl rand -base64 32

# On Windows PowerShell
$bytes = [byte[]]::new(32); [Security.Cryptography.RNGCryptoServiceProvider]::new().GetBytes($bytes); [Convert]::ToBase64String($bytes)
```

### Environment Mode
- `development` - verbose logging, stack traces in errors
- `production` - minimal logging, no stack traces

## Security Notes

⚠️ **IMPORTANT**: 
- Never use default values in production
- Keep your `.env` file secure and never commit it
- Rotate JWT_SECRET periodically
- Use strong, unique passwords for database
